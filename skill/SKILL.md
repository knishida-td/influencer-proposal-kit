# インフルエンサー商品提案書 生成スキル

## 概要

インフルエンサーに対して商品コラボを提案するための資料を自動生成する。
リサーチ → 商材企画 → 画像取得 → PPTX生成 → Google Slidesアップの全工程を実行。

## 起動条件

以下のいずれかで起動:
- 「○○さんに提案書作って」
- 「@xxx に○○を提案して」
- 「インフルエンサー提案資料」
- Instagram URLと商品カテゴリの指定

## 必要な入力（最低限）

| 入力 | 必須 | 例 |
|------|------|-----|
| インフルエンサー | 必須 | Instagram URL or 名前 |
| 商品カテゴリ | 任意 | 「ヘアオイル」「調味料」（未指定ならAIが提案） |
| 価格帯 | 任意 | 「3000-5000円」（未指定ならAIが設定） |

## 実行フロー

```
1. リサーチ
   ├─ WebSearchでインフルエンサー情報を取得
   │  - フォロワー数、年齢、ターゲット層
   │  - 過去のプロデュース商品・コラボ実績
   │  - コンテンツの方向性（プチプラ/高級/ママ向け等）
   └─ 商品カテゴリが未指定なら、プロフィールから最適な商材を企画

2. 商材企画（リサーチ結果に基づき決定）
   ├─ 商品コンセプト（なぜこの人にこの商品か）
   ├─ 主要スペック3点（素材/機能/技術の柱）
   ├─ 競合2ブランド + 比較スペック5項目
   ├─ 品質証明2点（認証/エビデンス）
   ├─ カラー/バリエーション3種
   ├─ 販売戦略3施策
   ├─ 価格帯 + 生産数 + 売上想定
   └─ スケジュール（企画→リリース）

3. 画像取得
   ├─ Pexelsから商品/ライフスタイル画像を8-10枚DL
   ├─ sipsで実ピクセル取得
   ├─ 目視確認: DLした画像が商品イメージと合うか必ず確認
   │  → 合わない場合は再検索してDLし直す
   └─ 画像ディレクトリ: /tmp/{proposal-name}-images/

4. スクリプト生成
   ├─ テンプレート構造（後述）に従いgenerate-{name}.jsを作成
   ├─ 保存先: /Users/nishidakeisuke/Projects/influencer-proposal-kit/
   └─ SlideKitデザインシステム準拠（pptx Skill参照）

5. PPTX生成 + QA
   ├─ node generate-{name}.js で生成
   ├─ PDF変換 → pdftoppmで画像化
   ├─ 重要スライドを目視確認（最低3枚: 表紙/素材/販売戦略）
   └─ 問題あれば修正して再生成

6. Google Slidesアップロード
   ├─ scripts/upload-to-gslides.sh でGoogle Drive APIに直接アップ
   │  （clasp認証トークン流用 → multipart upload → Google Slides自動変換）
   ├─ clasp未認証の場合は `open` でファイルを開いて手動案内
   └─ URLをユーザーに返却
```

## 13スライド テンプレート構造

全提案書は以下の固定構造。商材・ジャンルを問わず同じ。

| # | スライド | 内容 | レイアウト |
|---|---------|------|----------|
| 1 | 表紙 | 商品ビジュアル + タイトル | 左右に画像、中央にタイトル+赤黄ライン |
| 2 | コンセプト | なぜこの人にこの商品か | 見出し+本文+3箇条書き(赤バー) |
| 3 | 商品概要 | 写真+セールスポイント | 左に写真2枚、右に6箇条書き(赤丸) |
| 4 | 機能性/素材 | スペック詳細 3カラム | タイトル+サブ+写真大+説明文 x3列 |
| 5 | 品質証明 | 認証・エビデンス | DefBlock x2 + 右にSVGバッジ x2 |
| 6 | 競合比較 | スペック比較テーブル | 4列ヘッダー+5行データ |
| 7 | 生産背景 | 工場実績・製造品質 | DefBlock x2 + 右に工房写真 |
| 8 | デザイン詳細 | 商品デザインの仕様 | 左に商品写真大+右に箇条書き |
| 9 | バリエーション | カラー/フレーバー等 | 左に写真+右に3バリエーション |
| 10 | 販売戦略 | プロモーション計画 | 左に3施策+右に写真2枚 |
| 11 | スケジュール | タイムラインテーブル | 3列(時期/進行/詳細) x 6-7行 |
| 12 | 販売イメージ | 価格・数量・売上 | 3列(項目/内容/備考) x 8行 |
| 13 | エンドスライド | クロージング | 中央にタイトル+赤黄ライン |

## SlideKitデザインシステム（固定）

```javascript
const C = {
  bg: "111111", title: "FFFFFF", body: "E0E0E0", sub: "AAAAAA", muted: "666666",
  primary: "C9A96E", secondary: "E8D5B0", kmBg: "1F1A14", sep: "333333",
  divider: "333333", white: "FFFFFF",
};
const FONT = "Hiragino Kaku Gothic Pro W3";
```

ダーク背景+ゴールドアクセント。テーブル偶数行は `"1A1A1A"` を使用。

全スライド共通ヘルパー: `addHeader`, `addKeyMsg`, `addPageNum`, `centerY`, `addSep`, `addDefBlock`, `addImgFit`

## SVGバッジ生成

品質証明(スライド5)で使うSVGバッジは商材に応じてsharpで生成:
- ファッション: RDS認証, Vera Pelle, OEKO-TEX等
- ビューティー: COSMOS ORGANIC, ノンシリコン等
- 食品: GMP工場認証, 無添加認証等
- 一般: ISO認証, JIS規格等

## 画像のトンマナルール

ターゲット層に合った画像を必ず選ぶ:
- 30代女性向け → フェミニン、エレガント、明るいトーン
- プチプラ層 → カジュアル、親しみやすい、ピンク系背景
- 高級路線 → ダーク、洗練、ミニマル
- ママ向け → 家族感、実用的、暖かいトーン
- メンズ/ユニセックス → シンプル、無彩色、質感重視

ダウンロード後に必ず目視確認。商品イメージと合わない画像は差し替える。

## KeyMsg 28文字ルール

KeyMsg（各スライド下部の赤帯テキスト）は28全角文字以内に収める。
PowerPointで改行されない長さ。

## 注意事項

- 既存ファイルを上書きしない。別名(generate-xxx.js)で新規作成
- 画像はsipsで実ピクセルを取得しfitImageでアスペクト比維持
- sizing:coverは使わない
- SVGの`#`付きカラーコードはPptxGenJS本体で使わない（SVG内は可）
- 生成後は必ず画像化して目視QA（最低3スライド確認）
